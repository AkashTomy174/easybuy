# Git History Cleanup Script
# WARNING: This rewrites Git history! Use with caution.

## Prerequisites
# Install git-filter-repo: pip install git-filter-repo

## Step 1: Backup your repository
# Create a backup before running this!
# cp -r project project-backup

## Step 2: Create a file list to remove
# Save this list to a file called 'files-to-remove.txt':

ACTION_PLAN_NOW.md
AUTOCOMPLETE_IMPLEMENTATION.md
FILTERING_ENHANCEMENT.md
FIXES_SUMMARY.md
PLATFORM_COMPLETION_CHECKLIST.md
PROJECT_IMPROVEMENTS.md
REVIEW_ENHANCEMENTS_COMPLETE.md
REVIEW_ENHANCEMENTS.md
REVIEW_SYSTEM_FIXES.md
SETUP_COMPLETE.md
TROUBLESHOOT_SHIPPED.md
WHATSAPP_IMPLEMENTATION_SUMMARY.md
WHATSAPP_INTEGRATION_GUIDE.md
WHATSAPP_QUICK_START.md
WHATSAPP_README.md
DJANGO_ORM_STUDY_GUIDE.md
test_direct.py

test_order_items.py
test_order_notification.py

main.py
status_change_log.txt

## Step 3: Run git filter-repo (AFTER BACKUP!)
# git filter-repo --invert-paths --paths-from-file files-to-remove.txt

## Step 4: Force push to GitHub (DANGEROUS!)
# git push origin --force --all
# git push origin --force --tags

## Alternative: Using BFG Repo-Cleaner (Easier)
# 1. Download BFG: https://rtyley.github.io/bfg-repo-cleaner/
# 2. Run: java -jar bfg.jar --delete-files "*.md" --delete-files "test_*.py"
# 3. Run: git reflog expire --expire=now --all && git gc --prune=now --aggressive
# 4. Force push: git push origin --force --all

## ⚠️ IMPORTANT NOTES:
# 1. This will rewrite ALL commit hashes
# 2. Anyone who has cloned the repo will need to re-clone
# 3. Open pull requests will be affected
# 4. This cannot be undone easily
# 5. Make sure you have a backup!

## Recommended Approach:
# Since the files are already untracked, just leave them in history.
# They won't appear in future commits and won't clutter your working directory.
# This is the safest option for most projects.
